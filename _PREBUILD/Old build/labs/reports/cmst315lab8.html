Notes, Observations, and Questions
Day 1: Setting up a free-trial of Microsoft Azure on my university account, and exploring a little of its functionality.
Microsoft Azure is an internet-based cloud-computing service where one can create virtual servers, networks, machines, and much more without the use of hardware. It is a paid service that charges clients [that is, those who set up the virtual machines/networks/et caetera] based on storage use, uptime, and traffic. As it is an online service, clients can access it from any web browser and any computer provided they have internet connections.
As a student, all I had to do was follow a link provided by the professor to an Azure Dev Tools for Teaching, sign into my student account, and set up my Azure free trial within it. The intended goal was to use Azure’s online tutorials to set up a functioning virtual web server. My partner and I agreed to make our web servers Linux-based, after discovering that Azure supports a few Linux operating systems. As I proceeded through the lab, I only found two of the tutorial articles to be helpful toward my goal, and spent half of the time on my own or finding other helpful sources.
The article Tutorial: Create and Manage Linux VMs with the Azure CLI explained how to launch Azure Cloud Shell, a terminal component of Microsoft Azure, and use it to create resource groups and virtual machines, and connect to and manage those virtual machines (Nottingham et al., 2018). We each followed the tutorial to the letter, creating our VMs with Ubuntu Server 18.04 and secure-shelling into them (Fig. 1).
Day 2: Giving our virtual servers functionality -- or not
The next step towards turning our new servers into web servers was to install web server software onto them. My partner would install the Apache HTTP Server on his machine, and I would install NGINX onto mine. I worked with Apache before, and from experience it’s remarkably simple to set up, but I wanted to branch out.
The second of two articles that I found helpful, Quickstart: Create a Linux virtual machine in Azure with PowerShell, included a section on how to install the NGINX web server onto Ubuntu Server using apt (Nottingham et al., 2019)(Fig. 2). The tutorial seemed to suggest that implementing NGINX would be just as simple as implementing Apache. However, when I tried to view the default webpage on my server, the connection timed out and I was presented with an error page (Fig. 3).
Day 3: Outside of class -- what could have possibly gone wrong?
Because Azure is an online environment (and I have an individual account), I could examine my virtual machine and continue lab work outside of the classroom.
Attempts to access the server via URL were still unsuccessful, so I found a beginners’ guide for how to run the NGINX web server. Near the beginning of the article was a piece of information that seemed out of place for an instruction guide teaching how to run a piece of server software on a Linux machine: “To start nginx, run the executable file (Beginner’s Guide)”. This was unusual to me because while there is such a thing as an executable file, it typically has a .exe extension and programs with such extensions can only be run on a Windows operating system (Fig. 4). Thankfully I found another article that listed some terminal commands for interacting with NGINX, namely for starting, restarting, stopping or checking the status of the web server (Gite, 2016). A status check, and subsequent checks, for that matter, with sudo systemctl status nginx, consistently showed “nginx.service: Failed to parse PID from file /run/nginx.pid: Invalid argument” (Fig. 5). Attempts to connect to the site on the server still timed out.

Handout Questions
“What are some advantages and disadvantages of using cloud-based architecture?”
  • Advantages: The client doesn’t need to expend money and space on physical hardware like servers, cabling, switches and routers.

  • Disadvantages: The client is relying on the cloud-service provider to keep its server farms and data functional and secure.

“What type of things does a business need to consider before moving IT functions to the cloud?”
  • How much will cloud services cost?

  • What functionality does cloud services offer?

  • How does the provider of these cloud services keep clients’ data secure?

“Give some examples of some built-in configurations and services provided by Azure.”
  • Aside from virtual networking resources, Azure provides resources for storage, Internet of Things, security, databases, among others.


Conclusions
In this lab, I learned how to set up virtual resources on Microsoft Azure, a paid data-storage and cloud-service. I also got a fair amount of practice with the Linux command line and a brief peek at the inner workings of a Ubuntu Linux Server despite not successfully making a functional web server.
