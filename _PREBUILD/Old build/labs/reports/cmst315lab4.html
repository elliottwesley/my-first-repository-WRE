Notes, Observations, and Questions
Day 1 - Using NTFS as an Administrator
NTFS, or the NT File System, is a means of access control which allows administrators to permit or deny access of files or folders to users and/or groups of users.  It is based on the permissions granted to every user in a Windows-based network, regardless of whether the user is a local user or one created within the network (Active@ Data Recovery Software).  The goal for today was to create a folder on our virtual server’s desktop with sub-folders within that have different permissions via NTFS and try to access them from our virtual client.
Before we could create directories to access from the virtual client however, we had to create a small hierarchy of containers and groups to encompass our user accounts from the Active Directory lab.  In our virtual server’s Server Manager, we created our groups by first going into Active Directory Users and Computers and selecting our previously created domain [Domain1.local].  Then, under Action, and then New is a Group button.  This opens a settings box where the administrator can specify the type and name of the group.  In this case, we made a Global and Security group.  Right-clicking on the newly created group and selecting Properties brought up another settings box with multiple tabs.  The Members tab is where the administrator can assign users and other groups to be members of a group.  For our purposes, all four of our group’s users became members of this global group.
We then created a domain local group within our Domain1.local in similar fashion.  Instead of selecting Global in the group type, we selected the Domain local option, in addition to the Security option.  Our global group, with the users it encompasses, became the sole member of our domain-local group.
With our groups created, we then created a folder on the desktop of our server [called Home],  and created four directories within it [Test1, Test2, Test3, Test4]. To enable user accounts to access these folders on the virtual client, right-clicking on the Home folder and selecting Properties brings up a settings box.  Under Security is where the server’s administrator can select groups to allow to view or change the contents of the folder.  Selecting a group and clicking Edit allows the administrator to ‘share’ the directory with the group and dictate what actions the group will be permitted to do: read-only, read-and-write, or full-control.
Once we shared the folder to our domain-local group, we could access its contents from our client virtual machine.
