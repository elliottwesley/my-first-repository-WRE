Notes, Observations, and Questions
Objective 1 of 2 - Construct a simple network and program it for use in the later portion of the lab.
Everyone was split into three groups of two, and each group would build a similar network: each student in a group would have a computer on the end, connected to a switch, which is connected to a router in the center (See Figure 1).  Power cords would of course be linked to the appliances as needed.
The next step was to use Hyperterminal, a terminal emulator, to program the router by assigning IP addresses and standard subnets to each side of the network.  According to the instructions on the lab handout, we were to program one side to be a class C network and the other a class B network (See Figure 2).  Much like any terminal, programming a router requires use of a series of commands (See Figure 3).  The result was a small network in which one computer could reliably ping the other and receive a response.
Objective 2 of 2 - Learn how packet sniffing software works.
DISCLAIMER: Packet sniffing software can be classified as spy equipment.  Therefore, if you decide to make use of such software, ONLY use it on networks where you have permission to do so, namely your own.  You can be arrested for using this software to spy on other people’s computers/networks without proper authorization.
For the second component of this lab, we explored the world of cyber-surveillance by using Wireshark, one of many programs designed to sniff packets, discern what users are doing on a network, and detect intruders.  Such programs work by taking ‘snapshots’ of activity; the protocols associated with each packet can then be examined to find the source and intended destination of that packet and the request/contents inside (See Figure 4).  We tested this software by pinging the addresses of various websites.
