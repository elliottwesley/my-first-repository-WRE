Objective 2 of 2
The latter part of the lab centered around use of the Command Prompt application on the laboratory’s computers.  The ipconfig /all command (ifconfig on Macintosh and Linux computers) can be used here to find the computer’s physical (MAC) address and IP address.  Any device that potentially can use the internet (smartphones, tablets) has a MAC address and an IP address.  On some of these, the user need not open a terminal, and can find it in the settings application.
The ping command is used to get a reply from another address in the network or anywhere in the world and returns an amount of milliseconds based on how much time elapsed between the request and the response from the pinged server, thus verifying connectivity.  The next task involved use of the arp -d command, but attempts to do so by anyone in the laboratory failed because it required “elevation” or elevated permissions (darn security measures). The task would have been done like this:
The arp -a command would provide a list of all the other known addresses in the network.  To use the arp -d command would wipe all known addresses from this list.  Then, any pings to other addresses would add those addresses to the list, much like how a bridge or a switch functions in a network: it initially repeats data sent to it, but it learns the addresses of any computers that send data its way, and later on it can properly relay information sent to any of the now-known addresses.
The tracert command is like ping, except more in-depth.  Whereas ping simply confirms whether or not a computer can connect to another address (or server), tracert can be used to more effectively determine how far away another address (or server) is.  The command attempts to send four packets to the destination address through the fastest route possible in hops, or portions of the route to the destination.  Starting with one hop, and increasing by one with each failed attempt to get a reply, tracert attempts to contact the destination address.  When all four packets get a reply, shown by four consecutive pings and millisecond results, the connection is successful and the user learns how many hops away the destination address is [Fig. 3]. Generally, the more hops away an address is, the longer it takes to ping or connect to that address.

Handout Questions
“If we already had the TCP/IP model, why was the more complicated OSI model created?”
The purpose of the OSI model is to elaborate beyond simple network addresses and communication between each.  The OSI model includes additional layers for ensuring that data gets to where it needs to go without any missing portions and in a timely fashion, among other layers with other purposes.
“What does the abbreviation OSI stand for?  Who originated it? When?”
Open Systems Interconnection.  The origin for this concept was the merging of two separate projects from the late 1970’s: one from the International Organization for Standardization, and another from the International Telegraph and Telephone Consultative Committee.  The merging, in 1983, created The Basic Reference Model for Open Systems Interconnection. That same reference was published the next year. The idea of a seven-layer model comes from the work of then-Honeywell Information Services employee Charles Bachman (Wikipedia).
“What is the purpose of a router?  A switch? A hub? How are the devices similar?”  How are they different?”
A hub is a large repeater that multiple computers or devices can connect to, thus forming a network.  Any data that is sent into the hub is repeated across all connections.
A switch is similar to a hub in that multiple devices (and other switches) can hook up to it, but isolates each computer or device to its own collision domain.  It learns the IP addresses of all devices connected to it (in its network) and uses this information to help guide outgoing/incoming data where it needs to go.
A router is used to connect networks together and thus to transfer data between them.
Summary: All three devices are used in the formation of networks; multiple computers can hook up to a hub or switch (the latter effective for reducing collisions), and the router is used to unify whole networks thus forming even larger ones.
“What is the highest layer of operation for the following network devices: switch, hub, NIC, router, cable media, wireless access point, patch panel, repeater, bridge?”
Physical Layer (1) - Hub, cable media, patch panel, repeater
Data Link Layer (2) - Bridge, switch, wireless access point (WIP), network interface card (NIC)
Networking Layer (3) - Router
“What is another name for the physical address (assigned at the factory)?  How many bits does this address require?”
The Media Access Control (MAC) address.  It requires 48 bits of data.
“When two machines attempt to transmit simultaneously on the same media segment, both messages become garbled and unintelligible.  What is this condition called?”
A collision.
“What is the PDU unit of Layer 2?”
A frame.
“What protocol is used to dynamically assign IP addresses?  What is the purpose of DNS?”
Dynamic Host Configuration Protocol (DHCP).  DNS, or Domain Name System, is used to identify a unique address or server.
“What do the initials ARP stand for and what does it do?  What happens when you type the IP address of a web server into the address bar of your browser?”
Address Resolution Protocol.  It is used to discover the link layer, or MAC, address associated with a network layer, or IP, address.  When the user types an IP address into the URL or address bar, Address Resolution Protocol finds the link layer address associated with that IP and thus the website associated with that server and IP address.
“What protocol is used to translate a known layer 3 address into an unknown layer 2 address?”
Address Resolution Protocol.
“What are layer 3 devices called?  They use logical, or software encoded network addresses...What is the de facto standard for these logical/software/network addresses used today?  How many bits do they require?”
Layer three devices are called routers.  Like most other hubs and electronic devices, they use the IP and MAC addresses of the networks they are connected to, which require 32 bits (IPv4) and 48 bits respectively.
“What kind of information does the tracert command provide?”
In Command Prompt, the tracert command provides more detail as to how far another server/address is from the one the user is currently using.  It does this by attempting to send packets through the shortest path possible to the destination, from one router to another.  The results are shown as pings (response times in milliseconds) and in hops. Generally, the longer the response times, the more hops (and thus routers) that data has to go through to get to the destination.
