Notes, Observations, and Questions
Day 1 - Computer-side Policies
Today’s goals were to replicate the Idle-time lockout policy and Launching an application upon login examples in chapter 5 of Mastering Windows Group Policy.  The purpose of the idle-time lockout policy is to automatically lock the computer after a certain length of time if no input is provided from the user.  The latter setting opens certain applications when the user logs onto the workstation (Krause 2018).
As in previous labs, we implemented each example in its own group policy object.  Both of these were done as computer-side policies, meaning they were applied to the workstations and not the client accounts.  For our idle-time lockout policy, we set the machine inactivity limit to thirty seconds--not feasible for a professional network but okay for a laboratory setting--and linked it at the domain level.  We created a new organizational unit, entitled My Computers to contain our virtual client, created the application-launching policy to open the file explorer and the calculator applications whenever someone logged on (Figure 1), and linked the GPO to the new OU.  Logging into our virtual client with any of our client accounts caused the applications to open and the computer to lock after thirty seconds without user input, as expected.
We also discovered that the only way to see computer-side policies when printing the resultant set of policy in Command Prompt is to run it as administrator (Figure 2).  Otherwise, only the user-side policies will be shown (Buff, 2010).
Day 2 - Preferences and Drive Mappings
Today’s objectives were to examine the GPO’s User Configuration and Computer Configuration’s Preferences folders, and then replicate the Drive mappings and Creating a printer connection examples from chapter 6 of Mastering Windows Group Policy (Krause, 2018).
For our mapped drive GPO [which we named Drive maps], we found Drive Maps under Windows Settings and created a new mapped drive and specified its location as the Public Data folder in our domain, named it the same, and assigned M as the drive letter.  We linked to GPO at the root level.  After a quick gpupdate on the client computer, the mapped drive appeared under “Network Locations” in the file explorer.  For our printers GPO [that we named Printers], we found Printers under Control Panel Settings and added the shared printer that my colleagues set up in a previous lab [it’s path: \\WinServer1b\GTO] and assigned it as a default printer.  When we clicked the Print option inside a text file, the printer we added to the network did not appear the first few times.  Typing the path of the printer in the Control Panel’s file path bar then caused the printer to appear both in the list of network devices and as a location option when printing a document.  Strange...

Handout Questions
 
  • “How is a preference different from a policy setting?”

  Preferences are applied in a group policy update, but can be reverted by the user.  Policy settings cannot be reverted.
 
  • “How does the Administrative Template sections of GPOs provide flexibility to Group Policy?”

  Administrative Templates provide flexibility to Group Policy by allowing the administrator to create settings to manage applications not within the scope of the already massive quantity of potential settings that can be adjusted by a GPO.  Such applications are third-party software, not designed by Microsoft.
 
  • “Give examples of some of the settings an admin can control in the Computer Configuration section.”

  Examples of computer-side policies that can be set by an admin include the machine inactivity limit, the ability to automatically renew and update service certificates, and running certain scripts.
 
  • “Give examples of some of the settings an admin can control in the User Configuration section.”

Examples of computer-side policies that can be set by an admin include password-protecting the screen saver and launching specific applications when a user logs on to the network.

Conclusions
We learned the types of settings that can be configured in Group Policy; computer-side, user-side, and administrative template, and how they differ from one another.  We also learned how to view the computer-side policies in the Command Prompt and why printers are regarded as the most difficult piece of hardware to set up in a network.
